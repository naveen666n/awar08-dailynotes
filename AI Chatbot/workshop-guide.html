Deployment Guide · Build Your AI Assistant

Workshop Objective: Build and deploy a serverless AI assistant using Amazon Bedrock (Nova 2 Lite), Lambda, API Gateway and s3 Hosted Via Route53.

📋 Prerequisites & Setup

1 AWS Account Access & Region Selection
  • Ensure you have AWS Console access
  • Region: 🇮🇳 Asia Pacific Mumbai (ap-south-1)
  • Check you have permissions for Lambda, API Gateway, and Bedrock
Model Access: We'll use Amazon Nova 2 Lite via Global Inference Profile - automatically routes to the best available region for optimal performance. If you are using other region, you need to verify the model ID fro AWS bedrock servcie.
2 Create Workshop Directory

Create a folder on your computer called bedrock-workshop to store your files.

⚡ Step 1: Create Lambda Function

3 Navigate to Lambda Service
  • Open AWS Console
  • Search for "Lambda" in the services
  • Click on "Lambda" to open the service
4 Create New Function
  • Click "Create function"
  • Select "Author from scratch"
  • Function name: bedrock-ai-assistant
  • Runtime: Python 3.14
  • Architecture: x86_64
  • Click "Create function"
5 Add Lambda Function Code

In the Code source section, replace the default code with:

import json
import boto3
import logging

logger = logging.getLogger()
logger.setLevel(logging.INFO)

bedrock = boto3.client('bedrock-runtime', region_name='ap-south-1')

def lambda_handler(event, context):
    try:
        # Handle CORS preflight request (browser sends this before the real request)
        if event.get('httpMethod') == 'OPTIONS':
            return {
                'statusCode': 200,
                'headers': {
                    'Content-Type': 'application/json',
                    'Access-Control-Allow-Origin': '*',
                    'Access-Control-Allow-Methods': 'POST, OPTIONS',
                    'Access-Control-Allow-Headers': 'Content-Type'
                },
                'body': ''
            }

        # Parse the request body
        body = json.loads(event['body']) if event.get('body') else {}
        user_message = body.get('message', '')
        
        if not user_message:
            return {
                'statusCode': 400,
                'headers': {
                    'Content-Type': 'application/json',
                    'Access-Control-Allow-Origin': '*',
                    'Access-Control-Allow-Methods': 'POST, OPTIONS',
                    'Access-Control-Allow-Headers': 'Content-Type'
                },
                'body': json.dumps({'error': 'Message is required'})
            }
        
        # Prepare request for Nova 2 Lite
        request_body = {
            "system": [
                {
                    "text": "You are a helpful AI assistant built on AWS by Aviz Academy students. Answer clearly and simply."
                }
            ],
            "messages": [
                {
                    "role": "user",
                    "content": [{"text": user_message}]
                }
            ],
            "inferenceConfig": {
                "maxTokens": 1000,
                "temperature": 0.7
            }
        }
        
        # Call Bedrock with Nova 2 Lite profile
        response = bedrock.invoke_model(
            modelId='global.amazon.nova-2-lite-v1:0',
            body=json.dumps(request_body)
        )
        
        # Parse response
        response_body = json.loads(response['body'].read())
        ai_response = response_body['output']['message']['content'][0]['text']
        
        return {
            'statusCode': 200,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*',
                'Access-Control-Allow-Methods': 'POST, OPTIONS',
                'Access-Control-Allow-Headers': 'Content-Type'
            },
            'body': json.dumps({'response': ai_response})
        }
        
    except Exception as e:
        logger.error(f"Error: {str(e)}")
        return {
            'statusCode': 500,
            'headers': {
                'Content-Type': 'application/json',
                'Access-Control-Allow-Origin': '*',
                'Access-Control-Allow-Methods': 'POST, OPTIONS',
                'Access-Control-Allow-Headers': 'Content-Type'
            },
            'body': json.dumps({'error': 'Internal server error'})
        }
Model Used: Amazon Nova 2 Lite via Global Inference Profile - Routes automatically to the best available region.

Click "Deploy" to save the code.

6 Configure IAM Permissions
  • Go to the "Configuration" tab
  • Click "Permissions" in the left sidebar
  • Click on the execution role name (it will be a link)
  • This opens IAM in a new tab
  • Click "Add permissions" → "Attach policies"
  • Search for and select: AmazonBedrockFullAccess
  • Click "Add permissions"
7 Update Function Configuration
  • Go back to Lambda function
  • In "Configuration" → "General configuration"
  • Click "Edit"
  • Set Timeout to: 30 seconds
  • Click "Save"

🌐 Step 2: Create API Gateway

8 Navigate to API Gateway
  • Search for "API Gateway" in AWS Console
  • Click on "API Gateway" service
9 Create REST API
  • Click "Create API"
  • Under "REST API", click "Build"
  • Select "New API"
  • API name: bedrock-chat-api
  • Description: API for Bedrock AI Assistant
  • Endpoint Type: Regional
  • Security Policy: SecurityPolicy_TLS13_1_2_2021_06
  • Endpoint access mode: Basic
  • Click "Create API"
10 Create Resource
  • Click "/" and → "Create Resource"
  • Resource Path: /
  • Resource Name: chat
  • Enable CORS: ✅ Check this box
  • Click "Create Resource"
11 Create POST Method
  • Select the /chat resource
  • Under "Methods" section → "Create Method"
  • Select "POST" from dropdown
  • Click the checkmark ✓
  • Integration type: Lambda Function
  • Use Lambda Proxy integration: ✅ Check this box
  • Lambda Region: ap-south-1
  • Lambda Function: bedrock-ai-assistant
  • Click "Save"
  • Click "OK" to give API Gateway permission to invoke Lambda
12 Enable CORS
  • Select the /chat resource
  • Click "Actions" → "Enable CORS"
  • Access-Control-Allow-Methods: Select POST and OPTIONS
  • Access-Control-Allow-Headers: Content-Type,X-Amz-Date,Authorization,X-Api-Key,X-Amz-Security-Token
  • Access-Control-Allow-Origin: *
  • Click "save"
13 Deploy API
  • Click "Actions" → "Deploy API"
  • Deployment stage: [New Stage]
  • Stage name: dev
  • Stage description: Development stage
  • Click "Deploy"
  • 📝 Copy the Invoke URL - you'll need this for the web interface!
Important: Your API endpoint will look like: https://xxxxxxxxxx.execute-api.ap-south-1.amazonaws.com/dev/chat
Make sure to copy this URL - you'll need it in the next step!

        function formatText(text) {
            let formatted = text.replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;');
            formatted = formatted
                .replace(/^### (.*?)$/gm, '<h4 style="margin: 12px 0 8px 0; color: #0b5ed7; font-size: 16px;">$1</h4>')
                .replace(/^## (.*?)$/gm, '<h3 style="margin: 14px 0 10px 0; color: #0b5ed7; font-size: 18px;">$1</h3>')
                .replace(/^# (.*?)$/gm, '<h2 style="margin: 16px 0 12px 0; color: #0b5ed7; font-size: 20px;">$1</h2>')
                .replace(/\*\*(.*?)\*\*/g, '<strong style="color: #1f2937; font-weight: 600;">$1</strong>')
                .replace(/^\* (.*?)$/gm, '<li style="margin: 4px 0;">$1</li>')
                .replace(/^- (.*?)$/gm, '<li style="margin: 4px 0;">$1</li>')
                .replace(/\n\n+/g, '</p><p style="margin: 10px 0;">')
                .replace(/\n/g, ' ');
            formatted = formatted.replace(/(<li.*?<\/li>\s*)+/g, (match) => {
                return '<ul style="margin: 10px 0; padding-left: 20px; list-style-type: disc;">' + match + '</ul>';
            });
            formatted = '<p style="margin: 10px 0;">' + formatted + '</p>';
            return formatted.replace(/<p[^>]*>\s*<\/p>/g, '');
        }
        function addMessage(content, isUser) {

            const messageDiv = document.createElement('div');

        function showLoading() {

            loadingDiv.id = 'loading';
            loadingDiv.innerHTML = '<strong>AI Assistant</strong>🤔 Thinking...';

        function removeLoading() {
            const loading = document.getElementById('loading');
            if (loading) loading.remove();

        async function sendMessage() {
            const input = document.getElementById('messageInput');
            const sendButton = document.getElementById('sendButton');
            const message = input.value.trim();
            if (!message) return;
            addMessage(message, true);
            input.value = '';
            sendButton.disabled = true;
            showLoading();
            try {
                const response = await fetch(API_ENDPOINT, {
                    method: 'POST',
                    headers: { 'Content-Type': 'application/json' },
                    body: JSON.stringify({ message: message })
                });
                const data = await response.json();
                removeLoading();
                if (response.ok) {
                    addMessage(data.response, false);
                } else {
                    addMessage(`❌ Error: ${data.error || 'Something went wrong'}`, false);
                }
            } catch (error) {
                removeLoading();
                addMessage(`❌ Error: ${error.message}`, false);
            }
            sendButton.disabled = false;
            input.focus();
        }
        function handleKeyPress(event) {
            if (event.key === 'Enter') { sendMessage(); }
        }

🌐 Step 5: S3 Static Website Hosting with Route53

19 Create S3 Bucket
  • Navigate to S3 in AWS Console
  • Click "Create bucket"
  • Bucket name: your-domain-name.com (must match your domain)
  • Region: Asia Pacific Mumbai (ap-south-1)
  • Uncheck "Block all public access"
  • Acknowledge the warning about public access
  • Click "Create bucket"
Important: The bucket name must exactly match your domain name for Route53 alias to work properly.
20 Enable Static Website Hosting
  • Click on your newly created bucket
  • Go to "Properties" tab
  • Scroll down to "Static website hosting"
  • Click "Edit"
  • Select "Enable"
  • Hosting type: Host a static website
  • Index document: index.html
  • Error document: index.html (optional)
  • Click "Save changes"
  • 📝 Copy the Bucket website endpoint - you'll need this for testing
21 Configure Bucket Policy
  • Go to "Permissions" tab
  • Scroll to "Bucket policy"
  • Click "Edit"
  • Paste the following policy (replace your-bucket-name with your actual bucket name):
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::your-bucket-name/*"
        }
    ]
}
  • Click "Save changes"
22 Upload Website Files
  • Go to "Objects" tab
  • Click "Upload"
  • Click "Add files"
  • Select your index.html file
  • Click "Upload"
  • Wait for upload to complete
Test Your Website: Use the bucket website endpoint from Step 20 to verify your site is accessible.
23 Configure Route53 Hosted Zone
  • Navigate to Route53 in AWS Console
  • Click "Hosted zones" in the left sidebar
  • Select your domain's hosted zone (or create one if needed)
  • Click "Create record"
Prerequisites: You must have a registered domain and a hosted zone in Route53 before proceeding.
24 Create A Record for Root Domain
  • Record name: Leave blank (for root domain) or enter subdomain
  • Record type: A - Routes traffic to an IPv4 address
  • Enable "Alias" toggle
  • Route traffic to: Alias to S3 website endpoint
  • Choose region: Asia Pacific Mumbai (ap-south-1)
  • Choose S3 bucket: Select your bucket from the dropdown
  • Routing policy: Simple routing
  • Evaluate target health: No
  • Click "Create records"
25 Create CNAME Record for www Subdomain (Optional)
  • Click "Create record" again
  • Record name: www
  • Record type: CNAME
  • Value: your-domain-name.com (your root domain)
  • TTL: 300
  • Click "Create records"
Note: This step allows users to access your site with www.your-domain.com
26 Verify DNS Propagation
  • Wait 5-10 minutes for DNS changes to propagate
  • Open your browser and navigate to your domain
  • Verify your AI Assistant website loads correctly
  • Test the chat functionality
Success! Your AI Assistant is now live on your custom domain with S3 static hosting and Route53 DNS management.
