In linux everything starts from "/"

/ : Root Directory : Top level directory, the parent of all ither directories in Linux OS.

/root : The home directory for root user. (superuser)
windows: similar to C:\Users\Administrator

/home : Home directory of regular/non-root users. 
Windows: c:\users\<username>

/bin : Binary Executables : Contains all essential binary files used by all users (ex: ls, cp, mv).. 

/boot : Boot loader files : Contains the linux kernel, boot configs and its related files.

/dev : Device Files : Contains files that represent hardaware devices like hard disks. 
windows: device manager

/etc : Configuration files : Stores system wide configuration files (i.e; /etc/passwd, /etc/resolv.conf)
*Service management and system settings

/usr : user Programs : Contains uuser level programs and libs. 

/sbin : System binaries : contains system admin commands i.e; ifconfig, iptables
*Only root or superusers can execute these commands.

/var : Variable files : Contains data that changes frequently. (logs, databases, documentroot Path)

/lib & lib64 : Essential shared Library files.. 
* similaar to windows .dll files

/proc (Process information) : COntains system runtime information (CPU / RAM / processes)

/opt : Optional Software 
/mnt : Mount points : 
/media : removable media :

Storage devices : HDD & SSD*


EBS : Elastic Block Store : 

IOPS : Input and Output operations per Second

Block Size : 4 kb, 8 kb, 16 kb.
latency : The time taken to process a single I/O operation.
Throughput : At what rate data transimission happening.. mb/s, gb/s..

IOPS = throughput x 1024 / Blocksize

---

AWS EBS VOlume types: gp2, gp3, io1, io2, sc1, st1 and standard

SSD 		: gp2, gp3, io1, io2
HDD 		: sc1, st1
Magnetic 	: standard

root volume : VOlume contains OS : Without this volume, we cannt launch ec2 instance.

root volume : gp2, gp3, io1, io2 & standard
Addl volume : gp2, gp3, io1, io2, sc1, st1 and standard

---

gp2 and gp3 : General purpose SSD : we use this types for most common workloads.. 

gp2 : Min 1 GB.. Max 16 TB	(We can not configure iops count, we get it 1:3 ratio (1 gb vol = 3 IOPS with min of 100 IOPS)) - 16000 IOPS
gp3 : Min 1 GB.. Max 64 TB	(Min: 3000 IOPS, Max: 80000 IOPS.)

---

io1 and io2 : If any workload that requires specific IOPS count, we go with this.. 

io1 : Min 4 GB.. Max: 16 TB (1 : 50 Ratio)
io2 : Min 4 GB... Max: 64 TB.. (Min: 100 IOPS, Max: 200000 IOPS) 

---

HDD : sc1, st1 : bit older.. Less perf compared to SSD.. 

Throuthput Optimised HDD / st1 : suitable bigdata, data warehousing and log processing kind of workload.. 
Cold HDD / sc1 : Less performance than st1.. less cost also.. 

Min: 125 GB.. Max: 16 TB..

---

Standard (magnetic) : low cost storage solution.. less used.. min 1 gb .. max : 1TB
