Restart Policies: Control what happenes when a cotnainer stops or the host reboots.

docker run -d --restart=always --name mynginx -p 80:80

"always" or "unless-stopped" --> Mostly uses in Production environment.

always 			: Always restart, including on host reboot
unless-stopped 	: Similar to always, But not if manually stopped.
no				: Never restart (default)
on-failure		: Restart on failure if contianer exist tith error. 
on-failure:4	: Restart on failure, Max attempts to 4.

docker run -d -p 8080:80 --restart=always --name mynginx-withlimit --memory=256m --cpus=1 nginx

Host : COntainer uses the host machiene's IP and ports directly. No Virtual Network Layer.
--> No need for -p (port mapping) - Container port is the host port.


Bridge Mode : 
--> It will have its own virtual IP inside docker network
--> Required port mapping (-p 80:80 / -p 8080:80)
--> More isolated and secured
--> Works on Linux, mac and windows
--> One additional hop added, so, compared to host mode, small latency
--> Best used for enterprise and multi container appliations.

Host Mode: 
--> Shared host's IP and Port
--> port mapping not required as it uses host port.
-->  As this uses the host network, its Less isolated.
--> Works on Linux only
--> Sighltly faster as there is no NAT/Docker network.
--> System level tools or Monitoring agents.

FROM	: Sets the base image. Must be the first instruction in every Dockerfile.

FROM ubuntu:20.04
FROM python:3.9-slim
FROM node:latest
FROM alpine
FROM bookwarm

---

RUN 	: Executes commands during the build process. 

RUN apt-get update && apt-get install nginx -y
RUN dnf update && dnf install httpd -y

use && to combine multiple comamnds into single layer. 

----

WORKDIR		: Sets the working directory for all subsequent instructions (RUN, COPY, CMD)

WORKDIR /app
WORKDIR /usr/share/nginx

---

COPY	: Copies files/Directories from our hosy machine into the image. 

COPY <src-path> <container-dest-path>
COPY src/ /app
COPY . .

---

ADD		: Similar to COPY, But it also supports "Zip file & zip extracts"

ADD my_webcont.tar.gz /app

use COPY option, Unless you specifically need to extract a zip file. 

---

CMD		: Sets the default command to run when the container starts. CMD Gets overridden if you pass arguments while you are executing 'docker run'.

CMD ["nginx", "-g", "daemon off;"] 


FROM amazonlinux:2023
CMD ["echo", "hello from AVINASH"]		--> THis image build as "cmd-demo" and when we run it.. 

Not passing argument: 
docker run cmd-demo		--> "hello from AVINASH"

passing "date" comamnd while running the image
docker run cmd-demo	date --> Thu Jun 18 08:25:20 IST 2026
docker run cmd-demo	"hello from OVERWRITE for CMD" --> hello from OVERWRITE for CMD

---

ENTRYPOINT 	: Defines a comamnd that always runs using ENTRYPOINT, Even if arguments are passed with 'docker run'.

ENTRYPOINT ["echo", "hello from AVINASH"]

FROM amazonlinux:2023
ENTRYPOINT ["echo", "hello from AVINASH"]		--> THis image build as "entrypoint-demo" and when we run it.. 

Not passing argument: 
docker run entrypoint-demo		--> "hello from AVINASH"

passing "date" comamnd while running the image
docker run entrypoint-demo	"Docker for Students" --> hello from AVINASH Docker for Students

We can combine both "CMD" and "ENTRYPOINT"

FROM python:3.8

ENTRYPOINT["python"]
CMD["app.py"]

docker run python-demo	--> python app.py
docker run python-demo test.py	--> python test.py

-----

ENV			: Sets the environment variables available during build and at runtime.

ENV APP_ENV=production
ENV NAME=avinash

-----

LABEL		: Adds metadata to the image (author, version, description)

LABEL maintainer="avizway@gmail.com" version="1.0"

----

EXPOSE		: Document which port container listens on. DOES NOT actually publicj the port, We still need -p when running the container.

EXPOSE 8080

---

USER	: Sets the user (optionally group) for running the container. Improves security by avoiding root user.

---

VOLUME 	: Create a mount point for persistent or shared data

VOLUME /var/lib/mysql

---

HEALTHCHECK	: tells docker how to check if the contianer is healthy and still working properly.

---

SHELL	: Changes the default shell used for RUN instructions

SHELL ["/bin/bash", "-c"]

---

ARG		: We can define varables available only during the build process (not at runtime)

ARG VERSION=latest

STOPSIGNAL : instruction in a Dockerfile configures the specific system call signal that Docker sends to the container's main process (PID 1) to trigger a shutdown

STOPSIGNAL signal
